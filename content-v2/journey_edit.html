2. 行程创建接口

URL：/journey/pc/create

描述：用户创建新行程时触发

Request：

{
    // 选择任意字段创建行程
    "id": 0, 
    "title": "", 
    "cover_img": "", 
    "start_date": null, 
    "high_lights": null, 
    "notice": null, 
    "remark": "", 
    "day_ids": [ ], 
    "user_card": true, 
    "expense_desc": null, 
    "days": { }, 
    "price_type": null, 
    "price_currency": "EUR", 
    "price_info": null, 
    // 必填
    "modules_order": [
        "user_card", 
        "bright_point", 
        "google_map", 
        "day_detail", 
        "notice", 
        "price", 
        "fee"
    ]
}

参数说明：

出行提示
行程报价
费用说明
3个模块当days长度为0时候，不能添加
所以用户不能直接添加这3个模块
必须在添加一天后才能添加这3个模块

modules_order 字段，前台必传，标识了模块的顺序

Response：

{
  "code": 0,
  "id": 70703, // 行程id
  "msg": "成功",
  "timestamp": 1528271991
}

4. 行程信息修改保存

URL：/journey/pc/update-info

描述：行程保存

Request：

{
    "days": {
        "981751": {
            "cities": [
                {
                    "code": 3149488, 
                    "name": "澳门", 
                    "region": "", 
                    "gps": {
                        "lat": 22.198745, 
                        "lng": 113.54387299999999
                    }, 
                    "days": 3, 
                    "country_code": "MO", 
                    "country_name": "澳门"
                }
            ], 
            "content": [ ]
        }
    }, 
    "day_ids": [
        981751
    ], 
    // 必填
    "id": 70703
}

参数说明：

（price_info 报价有单独修改接口，不是通过该接口保存，详见行程报价菜单）
price_currency 和 price_type 还是通过该接口update
前端判断哪些字段进行了修改，将制定字段给后台
删除某天是_delete字段为true ，详见详情接口的描述

Response：

{
  "code": 0,
  "msg": "成功",
  "timestamp": 1528273060
}
